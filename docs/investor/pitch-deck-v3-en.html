How it works

  • The worker writes the ticket / comment in their native language
  • The system auto-translates to the viewer’s preferred_language
  • The Hungarian admin reads everything in Hungarian — no Google Translate copy-paste
  • Two-way: admin replies also appear in the worker’s language

Engineering detail

  • Model: claude-haiku-4-5-20251001
  • Prompt caching on the system prompt — ~90% input-cost reduction
  • 30-day DB cache (translation_cache table) — repeated strings are free
  • XML-tagged input — prompt-injection protection
  • Fallback: Google Translate (if Claude API unreachable)

🦄 Bruno AI — Context-Aware Onboarding Companion

An animated AI character that coaches users mid-workflow — especially new foreign workers

What Bruno does

  • Context-aware tips: sees the current page and offers targeted advice
  • Animated reactions: motivating feedback paired with the gamification engine
  • Speaks the viewer’s language: follows each user’s language preference
  • Non-intrusive: only appears when useful — new user, stuck flow, new feature

Why it matters

A Filipino or Ukrainian worker rarely gets a proper onboarding on day one. Bruno steps in: shows how to file a ticket, where to complete the pulse survey, what to do when a damage report lands on their screen.

Under the hood

  • components/Bruno/BrunoCharacter.jsx
  • 296 lines of CSS — full animation library
  • Admin entry point: pages/BrunoTest.jsx
  • Drop-in on every admin or mobile screen
  • Claude-powered answers when the user asks a question
Result: shorter onboarding, fewer HR tickets, higher adoption — especially for new workers with a language barrier.
